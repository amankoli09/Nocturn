Nocturn is a private negotiation protocol for prices that should stay hidden until a mutual agreement is reached. It enables two parties to cryptographically prove whether a buyer's maximum and a seller's minimum overlap — without revealing either limit to the chain, the platform, or each other.

This is not simply an escrow application with a privacy layer. It is a selective-disclosure smart contract built specifically to maintain information asymmetry during negotiation.

---

## The Problem with Public Negotiation

Negotiation breaks the moment one side is forced to reveal their true threshold. If the buyer reveals their absolute ceiling, the seller anchors higher. If the seller reveals their floor, the buyer pressures lower. Traditional blockchain applications cannot solve this because public ledgers make all state visible by default, exposing both parties' leverage.

## The Nocturn Solution

Nocturn leverages Midnight's selective disclosure model and zero-knowledge proofs to solve this. The protocol ensures that exactly what needs to be public becomes public, while private limits remain entirely off-chain in zero-knowledge witness space.

### Observable Privacy Guarantee

The core `matchOffer` circuit takes two private inputs:
- `buyerMax` — the buyer's maximum willingness to pay
- `sellerMin` — the seller's minimum acceptable price

These values enter the local ZK circuit to produce a SNARK proof, but they are **never serialised into the transaction body** and **never written to the ledger**.

The ledger receives only the cryptographic proof and the minimal state required for execution:

| Data Element | Visibility | Justification |
| :--- | :--- | :--- |
| **Negotiation ID** | 🟢 Public | Enables both sides to reference the unique agreement |
| **Status** | 🟢 Public | Indicates whether the deal is `Pending`, `Matched`, or `Failed` |
| **Agreed Price** | 🟡 Public (on match) | Required for transparent settlement when a deal is struck |
| **Buyer Maximum** | 🔴 Never Public | ZK witness — retained exclusively in browser memory |
| **Seller Minimum** | 🔴 Never Public | ZK witness — retained exclusively in browser memory |
| **Offer Gap** | 🔴 Never Public | Mathematically unreachable; cannot be derived from public state |

### How It Works Under the Hood

The SNARK generated by the local client proves the following statement:

> "I possess values `buyerMax` and `sellerMin` such that `buyerMax >= sellerMin`"

The verifier (the Midnight consensus node) validates the proof without ever seeing the inputs. If the proof passes, the node records the updated status and the settlement price on-chain. The **Privacy Receipt** component in the Nocturn UI provides an auditable breakdown of exactly what was disclosed during the transaction.

## Architecture

Nocturn is designed as a modern, full-stack DApp utilizing a monorepo structure. It separates the ZK circuit definitions, the TypeScript API wrapper, and the React frontend into distinct layers.

```mermaid
flowchart LR
  B[Buyer Client] --> UI[React UI\nui/]
  S[Seller Client] --> UI
  UI --> WC[WalletContext\nLace DApp Connector]
  WC --> LACE[Lace Wallet Extension]
  UI --> NC[NegotiationContext\napi/ layer]
  NC --> |deployContract\nfindDeployedContract| API[@nocturn/api]
  API --> |ZK witnesses\nbuyerMax, sellerMin| PROOF[Local Proof Server\nlocalhost:6300]
  PROOF --> |signed SNARK| NODE[Midnight Preprod Node]
  API --> |subscribe state$| IDX[Preprod Indexer]
  NODE --> L[Public Ledger State\nStatus + Settlement Price]
```

## Local Development & Setup

### Prerequisites

1. **Node.js** (v22 recommended)
2. **Docker Desktop** (required for the local ZK proof server)
3. **Lace Wallet Extension** configured for the Midnight Preprod network

### Wallet Configuration

1. Install [Lace Wallet](https://chromewebstore.google.com/detail/lace/gafhhkghbfjjkeiendhlofajokpaflmk).
2. Create or import a wallet and switch the network to **Midnight → Preprod**.
3. In the DApp settings, set the Proof server to **Local** (`http://localhost:6300`).
4. Fund your wallet with testnet tokens from the [Preprod Faucet](https://midnight-tmnight-preprod.nethermind.dev/).
5. Generate `tDUST` tokens inside Lace to cover transaction fees.

### Installation

Clone the repository and install the monorepo dependencies:

```bash
npm install
```

### Build Pipeline

Compile the Compact smart contract to generate the ZK proving keys and TypeScript bindings:

```bash
npm run compile
```

Build the API layer and typecheck the entire workspace:

```bash
npm run build
```

### Running the DApp Locally

1. **Start the local ZK Proof Server:**
   ```bash
   npm run proof:up
   ```

2. **Launch the Frontend:**
   ```bash
   npm run dev
   ```

Open `http://localhost:5173` in your browser. From the UI, you can deploy a fresh negotiation contract directly to the Preprod network and invite counterparties to join.

## Future Roadmap

Nocturn is a foundational primitive that generalises beyond simple two-party price negotiation. The selective-disclosure pattern implemented here can be extended for:

- **Automated Settlement:** Connecting the matched state directly to shielded token transfers.
- **Multi-round Negotiations:** Supporting counter-offers without exposing previous limits.
- **B2B Contracting:** Anonymous rate negotiations for freelance and corporate contracts.
- **Real Estate Offers:** Sealed-bid mechanisms where losing bids remain cryptographically hidden.
